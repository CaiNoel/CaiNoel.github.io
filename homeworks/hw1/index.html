Reading Responses:

  1. It was engaging to read about some of the techniques used to draw user attention towards specific features – the likes/retweets count, for example. It also make me curious about the sort of testing that companies such as AirBnB have to run in order to get a sense of whether, say, providing tabs for dates/guests actually drives usage.
  2. I really enjoyed the GIFs of each app's user interface, deconstructed. It gave some insight into the thought process that goes into analyzing (and, subsequently, creating) UI and menus. It was interesting to see how the presence (or lack thereof) of specific elements changed how the eye naturally observes the app...what becomes the focal point, and what the app is trying to drive you towards.
  3. By routinely analyzing other apps, you’ll develop your product thinking and get better at designing apps yourself.
    This really got me thinking about how to break down other user interfaces – UI is usually something we only notice when we're not supposed to – everybody knows when an interface sucks, but articulating what makes one work takes more practice. It's certainly something I'm going to pay more attention to when using apps.
  4. Probably an 8/10. My only criticism is that the article prompts a lot of questions about design intent ("why would they do this?") but I think it perhaps should've explored at least one answer to that. That might not be information the author has, however. Overall, a very solid article.
